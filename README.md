# growth-note
## サービス概要
美容技術職（主にネイリスト）の講師・生徒・オーナー間で、生徒の成長記録やフィードバックを一元管理できる、交換日記型の生徒管理Webサービスです。講習ごとの評価やコメントを蓄積し、生徒自身が過去の成長を振り返ることでモチベーション維持と技術向上をサポートします。また、講師・オーナー間の情報共有をスムーズにし、指導や育成の質を高めることを目的としています。

## このサービスへの思い・作りたい理由
私はネイリストとして現場で働いた後、ネイル講師として「ネイリストを目指す生徒」や「すでに現場に立っているがスキルアップを目指す生徒」の指導に携わってきました。
その中で特に大変だと感じていたのが、講習時間外に発生する情報整理やコミュニケーションコストです。具体的には以下のような課題がありました。
* 講習中に伝えた改善点を、生徒が翌日には忘れてしまう
* 生徒からの質問を、所属サロンのオーナーや別の担当者に確認し、再度生徒へ伝えるまでに時間がかかる
* 生徒一人ひとりの進捗や特徴をメモで管理しており、情報量が膨大になる
* 過去との比較がすぐにできず、成長の実感を共有しづらい
* 複数講師が関わる場合、報告フォーマットが統一されず読みづらい
* 生徒の進捗をオーナーには報告しても、生徒本人に十分伝わらず改善につながりにくい
現状ではLINEのグループトークを使って講師・生徒・オーナー間でやり取りをしていましたが、他のトークに埋もれて過去の報告を探すのに時間がかかるなど、運用に限界を感じていました。
また、オーナー側のゴールは「売上を上げられるネイリストを育てること」であることが多く、生徒自身が早い段階から自分の成長や将来の数字を意識できる環境が重要だと感じていました。
こうした現場経験から、講師・生徒・オーナーが同じ情報を共有し、生徒の成長を継続的に可視化できるサービスが必要だと考え、このサービスを企画しました。

## ユーザー層について
本サービスの主なユーザーは以下を想定しています。
* 技術講習を受けているネイリスト（生徒）
    * 技術習得の途中段階で、成長を実感しづらくモチベーションが下がりやすい層
* 生徒を指導する講師（社内・外部講師）
    * 複数の生徒を担当しており、情報整理やフィードバック共有に課題を感じている層
* 生徒が所属するサロンのオーナー（閲覧中心）
    * 生徒の成長過程や指導内容を把握し、育成や経営判断に活かしたい層
MVPでは特に生徒と講師を主役とし、 成長記録とフィードバックの体験価値を検証することを目的とします。

## ユーザー種別と権限（MVP時点/追記）
MVPでは以下のユーザー種別を想定しています。
* 講師（管理者）
    * 生徒の登録
    * フィードバックの作成・編集
* 生徒
    * 自身のフィードバック閲覧

## サービスの利用イメージ
1. 講習終了後、講師がその日の講習内容についてフィードバックを入力
    * 評価（○・△・×）
    * 課題やコメント
2. 生徒はアプリ上でフィードバックを確認し、過去の記録と比較する（一覧表示）
3. 自分の成長が可視化されることで、次回講習までの目標が明確になる
4. 講師は過去の履歴を確認しながら、継続的で一貫した指導ができる
これにより、生徒のモチベーション維持や技術定着をサポートし、結果として離職率の低下や育成効率の向上につながることを目指します。

## ユーザーの獲得について
美容業界は横のつながりが強く、口コミで情報が広まりやすい特徴があります。そのため、以下の方法を想定しています。
* SNSでの発信
* これまで関わってきたサロンオーナーや講師への紹介
* 実際に自身が関わる現場での利用・検証
まずは小規模で利用を開始し、フィードバックをもとに改善していく想定です。

## サービスの差別化ポイント・推しポイント
生徒管理ツール自体は、出席管理や予約管理などを中心に既存サービスが多く存在します。
しかし、本サービスは「管理」ではなく「成長の可視化と共有」に重点を置いています。
* 講習ごとの評価とコメントを蓄積し、生徒自身が成長を実感できる
* LINEなどのチャットに埋もれがちな情報を一元管理できる
* 講師ごとにバラつきがちな報告内容を、一定のフォーマットで整理できる
* 生徒本人・講師・オーナーが同じ情報を共有できる
特に、ネイリストデビュー前の練習段階でモチベーションを保つことは難しく、この段階を支援することが離職率低下につながると考えています。

## 機能候補
## MVPで実装予定の機能
* 生徒管理機能
    * 名前・経歴などの基本情報
* 日付ごとのフィードバック機能
    * ○・△・×の評価
    * 課題・コメント入力
* 過去のフィードバック閲覧機能
→ 生徒の成長を「記録・振り返り」できる、サービスの核となる体験を検証する。
## 本リリース以降に追加予定の機能
* 仕上がり写真の登録
* 売上・原価の記録、集計機能
* スケジュール管理
* 掲示板機能

## 使用する技術スタック
* フレームワーク：Ruby on Rails(学習内容をそのまま活かせ、開発速度が速いため。)
* データベース：PostgreSQL
* デプロイ先：Render
* 使用予定のライブラリ
    * Devise：講師・生徒など複数ユーザーの認証管理を行うため
    * Enum / Enumerize：評価（○・△・×）などのステータスを安全に管理するため
    * ActiveStorage：将来的な写真アップロード機能追加を見据えて採用

## サービスの導入対象について（追記）
本サービスのMVPでは、個人で活動している講師やフリーランスの技術指導者による導入を想定しています。
企業単位での導入を想定した場合、個人情報の取り扱い・権限管理の複雑化・セキュリティ要件の高度化など、検討・実装すべき項目が多くなり、MVPとしての検証が難しくなると考えました。
そのため、まずは『講師個人が自身の生徒管理・成長記録のために利用する』という最小構成でサービスの価値検証を行います。
将来的には、利用状況やフィードバックをもとに、
企業導入（サロン・スクール単位）への拡張も検討したいと考えています。

## 最後に
本サービスは、私自身が講師として現場で感じてきた課題をもとに企画しています。まずはMVPとして「生徒の成長を記録し、振り返れる体験」を小さく作り、実際に価値があるかを検証しながら、段階的に機能を拡張していきたいと考えています。
